Bulletins, collections and articles

What is a bulletin?

A bulletin gives headline numbers and commentary, alongside essential quality and methodology information.

It should be released with new data, and clearly link to more detailed content for users who want more than an overview.

Bulletins should focus on topics that we know users are interested in. If we know users are interested in a number of topics in a release, we can publish more than one bulletin in a "collection".

A bulletin should give users A bulletin should not contain
An overview of the main findings, and first commentary on newly released data In-depth commentary
Links to the new data, and to specific parts of the data which might be useful Deep technical analysis
A summary of the most important quality and methodology information, linking to more detailed documents where necessary Detailed explanations of upcoming changes, roadmaps, or transformation plans

Collections

What is a collection?

A collection always contains the following things:

  • at least one dataset
  • a summary (bulletin) page, explaining the most important things about the new data

Where there's a clear user need for more detail on a specific topic (gender pay gap, for example), a collection can contain more than one summary page.

An overview page contains commentary on the top-level findings from a collection.

Where more in-depth analysis is required, articles can be added to a collection.

What is an overview page?

User research found that users like the term overview for this type of page. As "Overview" and "Summary" can be used interchangeably, we're doing more research to find out whether this could be confusing.

Where a collection contains more than one summary (bulletin), an overview page summarises the most important information from across all the summary pages.

For example, the Crime collection may include summaries on Weapons, Fraud and Homicide.

The Crime overview would comment on overall Crime trends in the latest data, and might briefly explain the main causes of any changes (linking to the summary pages for more detail).

What is an article?

We are running more research to understand whether the word "article" means anything to users in this context.

An article is less structured than a summary, and allows for more detailed information on a subject. Typical articles might include:

  • deeper analysis of a topic that cannot fit in the summary template
  • analysis of the latest data alongside other data sources
  • more information about methodology or planned changes to a collection

As with any content on our website, there should be a clear user need for everything we publish. The length and detail of an article should be relative to the interest of our users. We will measure this using analytics and through user research.

An article should only be used when it is not possible to communicate all of the information users need in a summary page (because of the number of words required, or because the analysis crosses multiple summaries or collections, for example).

How to decide whether you need more pages?

Digital publishing will help you determine whether more pages would add value. We will use search data and user research to understand what users need. We may run extra research, or suggest small-scale experiments we can run to validate the need for a new page.

We will also consider:

  • whether users might expect to find information about this topic anywhere else (on another page in this collection, or elsewhere on the ONS website, for example)
  • how the user need relates to the time and effort required to create and publish extra content
  • can we communicate this using anything that already exists
  • does it need to be published on the day of release
  • whether this content is best written as an article or a bulletin (summary)

Once the page has been published, we will continually monitor its use to understand whether there is a continuing user need for this content.
